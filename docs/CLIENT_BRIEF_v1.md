# Краткий отчёт: аудит и подготовка — mnsk7-tools.pl

---

## Что обнаружили

Провели технический аудит сайта до начала работ.

**Хорошее:**
- Сайт работает, оплата через Przelewy24 подключена.
- Установлен Yoast SEO, robots.txt настроен, sitemap есть.
- Есть child-тема — кастомизации не потеряются при обновлениях.
- Caching (LiteSpeed) и Redis — хорошая база для скорости.

**Что мешает продавать:**
- На сайте одновременно установлено **4 плагина фильтрации** — они конфликтуют между собой, создают мусорные страницы в поиске Google и замедляют сайт. Нужно оставить один.
- **Изображения товаров** весят 3–4 МБ каждое (PNG без сжатия). Это прямая причина медленной загрузки, особенно на мобильных. Из 34 000 фото только 168 сконвертированы в современный формат WebP.
- Установлено **~64 плагина**, из которых минимум 10 — дубли одного и того же функционала (два вишлиста, два строителя страниц, два плагина платёжки, два пикселя Facebook и т.д.). Каждый лишний плагин замедляет сайт и создаёт риск конфликтов.
- **3–4 системы кеширования** работают одновременно — это создаёт риск, что корзина и чекаут могут кешироваться и ломаться.
- xmlrpc.php открыт — технический риск безопасности.

**Что нужно уточнить (пока не видели из-за отсутствия доступа к базе):**
- Заполнены ли атрибуты товаров: материал обработки, диаметр, хвостовик, покрытие, количество зубьев. Без этого невозможно сделать нормальный фильтр подбора фрезы.
- Структура описаний товаров: есть ли разделение «для чего подходит / параметры / совместимость».
- Прописаны ли alt-тексты у фото товаров (важно для SEO).

## Что будем делать дальше

**Сразу после того, как вы обозначите приоритеты и пожелания:**

1. **Дискавери** — зафиксируем цели, сегменты покупателей, боли и требования к сайту. На основе этого — чёткое ТЗ.

2. **Чистка** — уберём конфликтующие плагины (фильтры, дубли), настроим один кеш-слой, закроем xmlrpc. Это быстро и сразу улучшает стабильность.

3. **Изображения** — массовая конвертация в WebP. Ощутимый прирост скорости без редизайна.

4. **Атрибуты и каталог** — если атрибуты не заполнены, начнём с этого: без структуры данных ни фильтры, ни подборщик фрез, ни SEO страниц категорий не работают нормально.

5. **Тема и UX** — редизайн на staging, без риска для продакшена. Изменения переносятся на сайт через проверенный деплой-процесс.

6. **SEO** — структура категорий, шаблоны Title/H1/Description, schema, страницы-лендинги под запросы «фрезы по материалу/операции».
