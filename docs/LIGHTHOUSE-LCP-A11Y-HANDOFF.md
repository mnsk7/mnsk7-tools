# Lighthouse: LCP, блокирующие запросы, A11y — handoff

Краткий отчёт по правкам по результатам аудита Lighthouse (LCP, блокирующие ресурсы, контраст, доступность).

---

## 1. LCP — первый товар в сетке карточек

**Проблема:** LCP-элемент — первое изображение товара в списке. Lighthouse: «Требуется значение fetchpriority=high», не применять lazy к LCP.

**Сделано в теме:**

- **Сетка карточек** (mobile PLP, bestsellery, любой цикл `content-product`): в `functions.php` заменён вывод миниатюры в цикле — для **первого** товара в цикле передаются `loading="eager"` и `fetchpriority="high"` в `woocommerce_get_product_thumbnail()`. Остальные товары без изменений.
- **Таблица PLP** (desktop): в `content-product-table-row.php` уже было `loading="eager"` и `fetchpriority="high"` для первого ряда — без изменений.

**Плагины:** Если используется ленивая подгрузка изображений (например, подмена `src` на `data-lazy-src`), первый LCP-изображение лучше исключить из такой подгрузки в настройках плагина или через фильтр (если есть).

---

## 2. Блокирующие запросы (CSS/JS)

**Проблема:** Много CSS блокирует отрисовку (~151 KiB, ~9,6 с), что может ухудшать LCP/FCP.

**В зоне темы:**

- Критичный CSS хедера и promo bar уже инлайнятся в `header.php` (`#mnsk7-header-critical`).
- Тема подключает только `style.css` и `main.css`; порядок и зависимости не менялись.

**Рекомендации (вне темы):**

- **Плагины:** Отложенная загрузка или объединение CSS плагинов (UM, WooCommerce Blocks, PWA, Select2, Pickadate и т.д.) — через настройки плагинов или, например, **WP Rocket → Remove Unused CSS**, чтобы уменьшить объём блокирующего кода.
- **Критичный CSS:** При желании можно сузить блок `#mnsk7-header-critical` до минимума под first paint и подгружать остальное асинхронно (риск FOUC — оценить на staging).

## 3. Принудительная компоновка (reflow)

**Проблема:** `woocommerce/header-cart.min.js` запрашивает геометрию после изменения DOM.

**Решение:** Скрипт из ядра/плагина WooCommerce; в теме не правим. При необходимости — отложенная загрузка корзины в хедере через настройки или фильтры WooCommerce/кэширования.

---

## 4. Размер изображений (600×600 vs контейнер 307×307)

**Проблема:** Миниатюры 600×600 больше контейнера, возможна экономия трафика.

**Рекомендация:** Завести размер миниатюр ближе к отображаемому (например, 324 или 400 px по большей стороне) в **Настройки → Медиа** или через `add_image_size` и использовать его для `woocommerce_thumbnail` в цикле. Тема готова к смене размера; изменение делается в настройках/плагине, не в CSS темы.

---

## 5. A11y — внесённые правки

- **Ссылка «Moje konto» в хедере:** Добавлен `aria-label`: для гостя — «Moje konto / Zaloguj się», для авторизованного — «Moje konto: [имя]». Уникальный и понятный текст для скринридеров (`header.php`).
- **Кнопки «Dodaj do koszyka»:** Фон сменён на `var(--color-primary-dark)` для кнопок добавления в корзину в цикле и в общих стилях кнопок WooCommerce, чтобы контраст белого текста соответствовал WCAG 4.5:1 (`05-plp-cards.css`, `17-buttons.css`).

**Не в теме (плагины/контент):**

- **Порядок заголовков:** Секция Allegro Reviews (`h4.mnsk7-allegro-reviews__title`) — шорткод из плагина. Чтобы не было пропуска уровней (h2 → h4), в плагине шорткода стоит выводить, например, `h2` для заголовка секции или согласовать иерархию с остальной страницей.
- **Уникальность текста ссылок:** Элементы вроде «Dodaj do koszyka», «Zobacz wszystkie bestsellery» и т.д. — генерируются WooCommerce/блоками; при необходимости донастроить через шаблоны/фильтры WC или блоков.

## 6. Сдвиг макета (CLS)

**Проблема:** Элемент `.mnsk7-header__actions` (например, «0,00 zł 0 Produkt») даёт небольшой CLS.

**Рекомендация:** Зарезервировать минимальную высоту/пространство под блок корзины в critical CSS хедера или в `04-header.css`, чтобы после подгрузки контента корзины не было смещения. При необходимости можно вынести в отдельный мелкий pass.

## 8. Проверка

- **LCP:** Запустить Lighthouse по archive/sklep (mobile); LCP-элемент — первое изображение товара; в HTML у первого `<img>` должны быть `loading="eager"` и `fetchpriority="high"`.
- **Контраст:** Проверить кнопки «Dodaj do koszyka» в панели покрытия или расширением a11y — контраст белого по тёмно-синему ≥ 4.5:1.
- **Ссылка «Moje konto»:** В DevTools или скринридере убедиться, что у ссылки есть осмысленный aria-label.
