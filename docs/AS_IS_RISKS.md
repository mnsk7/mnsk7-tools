# AS-IS Risks — mnsk7-tools.pl

Риски при миграции на staging, редизайне темы и деплое.

---

## 🔴 Высокие риски

### R01 — Конфликт кеш-плагинов ломает staging и prod
- **Что:** LiteSpeed Cache + WP Rocket (?) + Seraphinite Accelerator одновременно.
- **Риск:** при деплое на staging кеш-плагин может закешировать корзину/чекаут → заказы не создаются, клиент не видит изменений.
- **Митигация:** перед деплоем — отключить все кроме одного. На staging: в wp-config добавить `define('WP_CACHE', false)` для отладки.

### R02 — 4 фильтр-плагина: при удалении 3 → SEO-потери
- **Что:** `filter-everything`, `woo-product-filter`, `woocommerce-products-filter`, `woof-by-category` — все активны.
- **Риск:** у каждого своя URL-структура для фильтров. Если один из них индексируется в Google, удаление → 404 на проиндексированных URL.
- **Митигация:** перед удалением — проверить в Search Console, какие URL фильтров есть в индексе. Настроить редиректы.

### R03 — PNG 3–4 МБ: смена темы не исправит скорость
- **Что:** ~34 000 файлов в uploads, топ — 3–4 МБ PNG.
- **Риск:** новая тема не ускорит LCP, пока не конвертированы изображения. Клиент будет думать, что «новая тема медленная».
- **Митигация:** до/параллельно с редизайном — массовая конвертация в WebP (через webp-uploads + регенерация или сторонний скрипт).

## 🟡 Средние риски

### R04 — Смена темы: потеря кастомизаций Customizer
- **Что:** текущие настройки (цвета, шрифты, виджеты) хранятся в БД как theme mods для `mnsk7-storefront` (child темы Storefront).
- **Риск:** при смене темы или переносе на staging — настройки не перенесутся автоматически.
- **Митигация:** перед сменой темы — экспорт Customizer настроек (плагин Customizer Export/Import).

### R05 — Два плагина Przelewy24: конфликт только при дубле одного шлюза
- **Что:** `przelewy24` + `woo-przelewy24` — возможно это обычная оплата и Raty (рассрочка), тогда оба нужны.
- **Риск:** если оба плагина регистрируют один и тот же gateway — дубли, двойные уведомления.
- **Митигация:** уточнить назначение каждого плагина; при дубле одного шлюза — оставить один плагин.

### R06 — Атрибуты товаров: фильтры могут не работать сразу
- **Что:** ASSUMPTION — атрибуты могут быть заполнены хаотично или не заполнены.
- **Риск:** новый фильтр-плагин (который выберем) не покажет ничего полезного, если атрибуты пустые.
- **Митигация:** аудит атрибутов ДО выбора и настройки фильтра.

### R07 — robots.txt `Disallow: /?` блокирует параметрные URL
- **Что:** в robots.txt стоит `Disallow: /?` — блокирует все URL с параметрами.
- **Риск:** если фильтры используют ?параметры, они не индексируются. Если это было сделано специально (против мусора), то при новом фильтре нужно пересмотреть правило.
- **Митигация:** проверить Search Console до и после смены фильтра.

## 🟢 Низкие риски

### R08 — Elementor + Beaver Builder: при удалении одного — «ломаются» страницы
- **Что:** если контент создан в Elementor, а Beaver Builder деактивировать — ок. Но если есть страницы в Beaver — сломаются.
- **Митигация:** проверить, есть ли страницы с Beaver-содержимым до деактивации.

### R09 — ultimate-member + profile-builder: пользовательские поля в БД
- **Что:** при деактивации одного из них — кастомные поля пользователей могут «исчезнуть» из интерфейса.
- **Митигация:** выяснить, какой плагин реально используется для регистрации/кабинета.

### R10 — media-cleaner: риск удалить нужные файлы
- **Что:** плагин установлен, но использование неизвестно.
- **Риск:** если запустить очистку — удалит файлы, не привязанные к постам (но используемые в CSS/виджетах).
- **Митигация:** перед запуском media-cleaner — полный backup uploads.
