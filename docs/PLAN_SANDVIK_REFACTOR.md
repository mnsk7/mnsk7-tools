## 1. Плагины — что убрать / оставить

### Убрать (фильтрация/таблица — делать своим кодом в теме или mu-plugin)

- **Product Filter for WooCommerce by WBW** — заменить кастомной фильтрацией (чипсы по атрибутам/категориям).
- **Product Table and List Builder for WooCommerce Lite** — заменить своим шаблоном архива с таблицей (см. п. 4).

После удаления плагинов:
- Фильтры: выводить чипсы (категории, атрибуты) в шаблоне архива, фильтрация через GET-параметры или AJAX в теме/mu-plugin.
- Таблица: свой шаблон `archive-product.php` (или части) с разметкой `<table>` и строками товаров.

### Оставить (без изменений, если не мешают)

- Discount Rules, Flexible Checkout Fields, Furgonetka, Przelewy24, WooCommerce Tax, GTM4WP, Yoast, WPCode, и т.д. — по необходимости.
- **WooCommerce Load More Products** — можно оставить или позже заменить своей пагинацией/«подгрузкой» в таблице.

## 2. Футер и отрисовка (переход tech-storefront → Storefront)

**Проблема:** раньше тема была tech-storefront (parent: best-shop), сейчас — mnsk7-storefront (parent: Storefront). В файлах дочерней темы всё есть, но футер/страница отрисовываются плохо.

**Что проверить:**
- Родитель Storefront добавляет обёртки (`#content`, возможно `.content-area`). В `footer.php` дочерней темы закрываются два `</div>` — они должны соответствовать обёрткам, открытым в header и в WooCommerce (см. `woocommerce_before_main_content` в Storefront).
- Если на сервере не установлен родительский Storefront или подключается другая версия — обёрток может быть больше/меньше, футер «уезжает» или сжимается.

**Сделано в теме:**
- Футер вынесен в свой блок `.mnsk7-site-footer-block` и нижнюю полосу `.mnsk7-footer-bottom`.
- В CSS заданы `width: 100%`, `box-sizing`, подавление `.site-info` родителя.
- Класс `site-footer--mnsk7` для переопределения фона и отступов.

Если футер всё ещё «ублюдский»:
- В браузере (DevTools) посмотреть, внутри какого контейнера рендерится `<footer id="colophon">` и какие у него вычисленные стили (max-width, margin).
- При необходимости добавить в дочернюю тему переопределения для `#colophon` и родительских контейнеров (например, чтобы контент выше футера не ограничивал ширину футера).

## 4. Таблица в категориях + чипсы

**Цель:** в каждой категории товаров и на страницах tag-produkty (или аналог) показывать:
- чипсы фильтров (подкатегории, атрибуты);
- таблицу товаров как на [Sandvik taxonomy](https://www.sandvik.coromant.com/pl-pl/tools?q=*&taxonomy=INS%2FINSI).

**Реализация:**
- Шаблон: переопределение в теме `woocommerce/archive-product.php` (или только контента архива) для вывода:
  - заголовок категории/тега;
  - строка чипсов: активная категория/тег + атрибуты (например диаметр, материал) — ссылки или кнопки, ведущие на тот же архив с изменёнными GET-параметрами;
  - таблица: `<table class="mnsk7-product-table">` с колонками (фото, название, артикул, атрибуты, цена, действие).
- Данные: цикл по `$wp_query` (как сейчас в архиве), но разметка — строки `<tr>` вместо сетки карточек.
- Фильтрация: при клике по чипсу — переход на URL с `?filter_attr=...` или по таксономии; обрабатывать в `functions.php` или mu-plugin через `pre_get_posts` / WooCommerce query.

**Файлы для правок/создания:**
- `wp-content/themes/mnsk7-storefront/woocommerce/archive-product.php` — условие: выводить таблицу на страницах категорий и тегов.
- Опционально: `content-product-table.php` — один ряд таблицы (фото, название, артикул, цена, кнопка).
- CSS: `parts/24-plp-table.css` (или раздел в существующем) — стили таблицы и чипсов.
- Логика фильтров: хуки в `functions.php` или в mu-plugin (например `mnsk7-tools.php`).
