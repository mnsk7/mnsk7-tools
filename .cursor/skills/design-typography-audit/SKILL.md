# Аудит дизайна и типографики

Проводить аудит в три этапа: **шрифты**, **отступы и обводки**, **код vs реальность**. Результат — структурированный отчёт по шаблону из reference.

## Декомпозиция

### 1. Аудит шрифтов (типографика)

**В коде:**
- Найти все вхождения `font-family` в `wp-content/themes/mnsk7-storefront/assets/css/` (grep по `font-family`).
- Проверить: используется ли везде `var(--font-base)` или токен; нет ли жёстко заданных `"Arial"`, `sans-serif` без var, других `@font-face`.
- Токены шрифта и размеров: `01-tokens.css` — `--font-base`, `--fs-xs` … `--fs-hero`.

**Критерий:** один системный шрифт (Inter через `--font-base`); размеры — только через `--fs-*`.

**В отчёте:** список файлов и селекторов с отличным от `var(--font-base)` значением; рекомендации заменить на токены.

---

### 2. Аудит отступов и обводок

**В коде:**
- Отступы: grep по `margin`, `padding`, `gap` — есть ли «магические» значения (`16px`, `24px`, `1rem` и т.д.) вместо токенов `--space-*` (4, 8, 12, 16, 24, 32, 40, 48, 64).
- Обводки: `border`, `border-radius` — должны идти через `--color-border`, `--color-border-strong`, `--r-sm`, `--r-md`, `--r-lg`.
- Сравнить однотипные блоки (карточки, кнопки, секции): одинаковые ли значения отступов/радиусов в коде.

**Критерий:** повторяющиеся отступы/радиусы заданы токенами; однотипные UI-блоки используют одни и те же токены.

**В отчёте:** перечень селекторов с «магическими» значениями; таблица по типам блоков (какие отступы/радиусы где заданы).

---

### 3. Код vs реальность

**В коде:** что задано в CSS (какие переменные, селекторы, каскад, медиа-запросы).

**В браузере:**
- Открыть нужные страницы (главная, каталог, товар, корзина).
- DevTools → Elements → выбрать элемент → вкладка Computed. Сверить: `font-family`, `font-size`, `line-height`, `margin`, `padding`, `border`, `border-radius`.
- Если значение отличается от ожидаемого — проверить специфичность, `!important`, переопределения в других частях CSS.

**Критерий:** вычисленные стили совпадают с задуманными токенами; расхождения объяснены (какой правило перебивает).

**В отчёте:** для выборочных ключевых элементов — «ожидалось / фактически»; при расхождении — селектор и файл, которые переопределяют.

---

## Порядок выполнения

1. **Шрифты:** grep по CSS → заполнить раздел «Типографика» отчёта.
2. **Отступы/обводки:** grep по margin/padding/border/gap/radius → заполнить раздел «Отступы и обводки».
3. **Код vs реальность:** при возможности — проверить в браузере ключевые блоки и добавить раздел «Сверка с отрисовкой».

Использовать шаблон отчёта из [reference.md](reference.md).
