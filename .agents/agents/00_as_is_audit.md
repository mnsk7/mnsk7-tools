## Что делать

### 1) Составить список важных страниц

- Главная
- Категория
- Карточка товара
- Корзина
- Чекаут
- Аккаунт
- Поиск
- Служебные (доставка / возврат / контакты)

### 2) Проверить и зафиксировать (чеклист)

**A) Каталог / контент**
- Есть ли нормальные атрибуты: материал / операция / Ø / хвостовик / покрытие / зубья
- Качество названий и SKU
- Структура описаний (польза vs параметры)
- Фото: alt/title, единый стиль, количество, размер/вес

**B) UX / конверсия**
- CTA, наличие, цена, доставка, возврат рядом с CTA
- Фильтры/сортировки: полезность и скорость
- Мобильный UX
- Доверие (политики, гарантия, отзывы)

**C) SEO**
- Title/H1 на категориях и товарах
- Дубликаты, тонкие страницы (фильтры)
- Индексация (robots.txt, sitemap)
- Schema: Product / BreadcrumbList / FAQ

**D) Performance**
- Тяжёлые картинки, источники CLS/LCP
- Кэширование (исключения для cart/checkout)

**E) Tech**
- Тема: кастом/готовая, overrides Woo
- Список плагинов (что критично, что мусор)
- Есть ли child-theme
- Кастом код и где он живёт
- Безопасность: роли, xmlrpc, бэкапы

### 3) Выходные документы

- **`/docs/AS_IS_AUDIT.md`** — структурированный отчёт
- **`/docs/AS_IS_BACKLOG.md`** — приоритизированные улучшения
- **`/docs/AS_IS_RISKS.md`** — риски миграции / деплоя / SEO
